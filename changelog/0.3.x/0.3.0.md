# 0.3.0 — 2026-06-08

## Added

- **`openfda_dataframe_query`** — read-only SQL (`SELECT`, `GROUP BY`, aggregates) over a result set staged on a DataCanvas. Requires `CANVAS_PROVIDER_TYPE=duckdb`. ([#1](https://github.com/cyanheads/openfda-mcp-server/issues/1))
- **`openfda_dataframe_describe`** — lists tables and column schemas staged on a DataCanvas. ([#1](https://github.com/cyanheads/openfda-mcp-server/issues/1))
- **DataCanvas spillover on 8 search tools** — `openfda_search_adverse_events`, `openfda_search_recalls`, `openfda_search_drug_approvals`, `openfda_search_device_clearances`, `openfda_search_animal_events`, `openfda_search_drug_shortages`, `openfda_search_tobacco_reports`, `openfda_lookup_ndc` all accept an optional `canvas_id` input and return `canvas_id`, `canvas_table`, `spilled`, and `truncated` output fields when canvas is enabled. Opt-in via `CANVAS_PROVIDER_TYPE=duckdb`; when unset, search tools are byte-unchanged. ([#1](https://github.com/cyanheads/openfda-mcp-server/issues/1))
- **`@duckdb/node-api` `^1.5.3-r.3`** added as an optional DuckDB canvas backend dependency (lazy-loaded; no impact when canvas is disabled).
