# 0.3.1 — 2026-06-12

## Added

- **`openfda_count_values` truncation disclosure** — when the term list hits the `limit`, enrichment now carries `truncated`, `shown`, `cap`, and `truncationCeiling` (the count of the lowest-ranked term returned) so callers know more distinct values may exist.
- **`openfda_get_drug_label` truncation disclosure** — when `skip + results < total`, enrichment carries `truncated`, `shown`, and `cap` with a guidance hint to page with `skip`.
- **`scripts/clean-mcpb.ts`** — post-pack bundle cleaner wired into `bundle`: runs `mcpb clean`, then strips dependency-shipped agent-doc entries (`skills/`, `.claude/`, `.agents/`, stray `SKILL.md`) nested under `node_modules/` that root-anchored `.mcpbignore` patterns cannot reach.
- **Dockerfile `HEALTHCHECK`** — bun-native `fetch` against `/healthz` (the slim image ships no curl/wget); `APP_VERSION` build arg stamps the `org.opencontainers.image.version` OCI label.

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.21 → ^0.10.6`.
- **`src/index.ts`** — `createApp()` now sets `name` and `title` explicitly to `openfda-mcp-server` (display identity is the machine name on every surface; keeps the scoped npm name out of the served `server_name`).
- **`openfda_dataframe_query`** — canvas queries now pass `denySystemCatalogs: true`, blocking SQL access to DuckDB system catalogs.
- **`scripts/lint-packaging.ts`** — added bundle-content guards (dev dirs excluded at root, `.mcpbignore` patterns anchored so they don't strip runtime `node_modules/**` paths, built `.mcpb` carries zero `node_modules` agent-doc entries) and identity checks (`createApp()` `name`/`title` and manifest `display_name` must equal the unscoped package name).
- **`scripts/check-framework-antipatterns.ts`** — added rule 4 flagging `z.coerce.boolean()` on env flags (`Boolean("false")` is `true`; use `z.stringbool()`); comment lines are skipped so docs naming the pattern don't false-positive.
- **`.mcpbignore`** — dev-directory and config patterns anchored with a leading `/` so they match only at bundle root; added `/.agents/` and `/skills/`.
- **Skills** synced from framework `0.10.6`; `techniques` skill added.

## Dependencies

- `@cyanheads/mcp-ts-core` `^0.9.21 → ^0.10.6`
- `@types/node` `^25.9.2 → ^25.9.3`
