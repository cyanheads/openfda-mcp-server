# 0.1.17 — 2026-05-30

## Added

- **`openfda_search_animal_events`** — new tool querying `animalandveterinary/event` (1.3M+ records): adverse event reports for veterinary drugs and devices including animal species, breed, active ingredients, and reaction outcomes. ([#2](https://github.com/cyanheads/openfda-mcp-server/issues/2))
- **`openfda_search_tobacco_reports`** — new tool querying `tobacco/problem`: problem reports for tobacco products covering product type, problem description, and health effects. ([#2](https://github.com/cyanheads/openfda-mcp-server/issues/2))
- **`tobacco/problem`** endpoint added to `openfda_count`'s supported endpoints list.
- **Typed `errors[]` contracts** on all 8 tools (the 6 previously untyped, plus the 2 new tools): `rate_limited`, `upstream_error`, `query_error`, and `pagination_limit_reached` reasons declared with recovery hints surfaced in `content[]` via `ctx.recoveryFor()` at each service throw site. ([#5](https://github.com/cyanheads/openfda-mcp-server/issues/5))

## Fixed

- **HTTP 401** now throws `McpError(Unauthorized)` with `data.reason: 'unauthorized'` — non-retryable, operator-actionable; previously fell through to an opaque `InternalError`. ([#6](https://github.com/cyanheads/openfda-mcp-server/issues/6))
- **HTTP 403** now throws `McpError(Forbidden)` with `data.reason: 'forbidden'` — non-retryable; previously same opaque fallback. ([#6](https://github.com/cyanheads/openfda-mcp-server/issues/6))
