# 0.1.12 — 2026-05-08

## Added

- `emptyResultMessage(skip, baseHint)` in `src/mcp-server/tools/format-utils.ts` — emits a paginated-past-end hint when `skip > 0`, otherwise the base hint. Adopted by all six search/lookup tools so callers can tell `"no records match"` from `"ran past the result window"` (openFDA returns the same 404 for both).
- `OpenFdaService.lastUpdatedByEndpoint` cache — captures `meta.last_updated` from each successful response and reuses it on subsequent 404s for the same endpoint, replacing the stub `'unknown'` value. 404 responses also now preserve the request's `skip`/`limit` in `meta` instead of zeroing them.
- Two tests in `tests/services/openfda/openfda-service.test.ts` covering the new 404 behavior (skip/limit preservation and cached `lastUpdated` fallback).

## Changed

- `formatRemainingFields()` — new internal `renderValue()` helper formats string arrays as comma-joined values and flattens single-level objects to `key=value; key=value` pairs, falling back to JSON only for deeper structures. Empty containers now skip cleanly.
- `openfda_search_adverse_events.format()` — drug section skips FDA workflow timestamps and format codes (`safetyreportversion`, `transmissiondate*`, `receivedateformat`, `receiptdate*`, `reporttype`, `fulfillexpeditecriteria`, `duplicate`) so the clinical view isn't buried in metadata.
- `openfda_search_device_clearances.format()` — `advisory_committee` (the bare code) joins the rendered set alongside `advisory_committee_description` to avoid duplicate rendering.
- Tool descriptions:
  - All six tools that accept `sort` — corrected the sort-failure note from `"Unrecognized fields are silently ignored"` to `"Invalid or non-sortable fields cause a query error"`. Matches actual openFDA behavior.
  - `openfda_count`, `openfda_lookup_ndc`, `openfda_search_drug_approvals` — added cross-tool pairing hints (lookup_ndc → get_drug_label, search_drug_approvals → get_drug_label/count, count → search/get tools) so the LLM knows which companion to reach for.
  - `openfda_search_recalls.search` — classification example expanded to enumerate `Class I`, `Class II`, `Class III`.
  - `openfda_get_drug_label.search` — `set_id` example clarified as the SPL UUID returned in earlier results.
  - Output `results` schemas across `get_drug_label`, `lookup_ndc`, `search_device_clearances`, `search_drug_approvals`, `search_recalls` — replaced one-line summaries with concrete field inventories so callers don't have to call to discover the shape.
  - Empty-result `message` descriptions tightened to match the new behavior.
