# 0.1.16 — 2026-05-30

## Changed

- **`openfda_search_adverse_events`, `openfda_search_device_clearances`, `openfda_search_drug_approvals`, `openfda_search_recalls`, `openfda_count`** — each tool now emits a typed `enrichment` block via `ctx.enrich`; query echo (`effectiveQuery` / `echo`), result total (`totalResults` / `termCount`), and empty-result guidance (`notice`) reach both the `structuredContent` JSON channel and the `content[]` markdown trailer.
- **`openfda_get_drug_label`, `openfda_lookup_ndc`** — same enrichment adoption; the `message` output key that previously carried empty-result text has been replaced by `enrichment.notice` (value and guidance preserved, field moved to the enrichment block).

  > **structuredContent note:** the `message` key in the tool output schema has been removed; the same guidance now surfaces as `enrichment.notice`.

- **`@cyanheads/mcp-ts-core`** ^0.9.13 → ^0.9.16
