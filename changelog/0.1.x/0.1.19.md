# 0.1.19 — 2026-06-04

## Added

- **`openfda_search_drug_shortages`** — new tool over the `drug/shortages` endpoint ([#8](https://github.com/cyanheads/openfda-mcp-server/issues/8)). Returns shortage status, availability, therapeutic category, dosage form, manufacturer, and timeline. The `openfda` block carries `brand_name`, `product_ndc`, and `rxcui` for chaining into `openfda_get_drug_label` or `openfda_lookup_ndc`. Same `search` / `sort` / `limit` / `skip` contract as the other search tools.
- **`openfda_describe_fields`** — new tool that returns searchable field paths for any of 15 cataloged openFDA endpoints, grouped by category with data type and description ([#9](https://github.com/cyanheads/openfda-mcp-server/issues/9)). Covers all major endpoints (`drug/event`, `drug/label`, `drug/shortages`, `drug/drugsfda`, `drug/ndc`, `drug/enforcement`, `food/event`, `food/enforcement`, `device/event`, `device/510k`, `device/pma`, `device/recall`, `device/enforcement`, `animalandveterinary/event`, `tobacco/problem`).
- **`field-catalog.ts`** — static catalog of ~200 searchable field paths, shared by `openfda_describe_fields` and the reactive notice enrichment.

## Changed

- **Reactive field hints** — all 9 search tools (`search_adverse_events`, `search_recalls`, `search_drug_approvals`, `search_device_clearances`, `search_animal_events`, `search_tobacco_reports`, `search_drug_shortages`, `get_drug_label`, `lookup_ndc`) now append the top 6 field paths for the queried endpoint to the `notice` enrichment when results are empty, pointing agents at `openfda_describe_fields` for the full list ([#9](https://github.com/cyanheads/openfda-mcp-server/issues/9)).
