# 0.1.13 — 2026-05-16

## Added

- **`createApp({ instructions })`** in `src/index.ts` — server-level orientation text included on every `initialize` response. Spec-compliant clients SHOULD forward it to the model as session-level system context. Lists the openFDA endpoint coverage, query-string syntax (dotted paths joined by AND/OR with double-quoted phrases), and the `openfda.*` cross-product prefix so the model picks the right tool without leaking that guidance into every tool description.

## Changed

- **Framework upgrade** — `@cyanheads/mcp-ts-core` `^0.8.19 → ^0.9.1`. Notable inherited changes: Cloudflare Workers now boot under `nodejs_compat`; cross-vendor JSON Schema portability lint family (`schema-format-portability` and friends) — this server emits no `z.url()`/`z.cuid()`/etc., so no schema changes were required; definition linting moved to build-time only (`bun run lint:mcp` / `bun run devcheck`), so cold-start is faster and new lint rules are additive going forward. SSRF DNS validation and IPv6 blocklist coverage harden the `OpenFdaService` upstream fetch path.
- **Dev dependency bumps:**
  - `@biomejs/biome` `^2.4.14 → ^2.4.15`
  - `@types/node` `^25.6.2 → ^25.8.0`
  - `@vitest/coverage-istanbul` `^4.1.5 → ^4.1.6`
  - `vitest` `^4.1.5 → ^4.1.6`
- **`scripts/devcheck.ts` `bun outdated` parser fix** — `bun outdated` emits markdown-style rows (`| pkg | … |`) with an empty leading cell, so `split('|')[0]` was always empty and the allowlist never matched. Reads index `[1]` now and strips the trailing `(dev|peer|prod|optional)` workspace marker so bare package names match the allowlist.
- **Changelog summary cap raised 250 → 350 chars** — `SUMMARY_MAX_LENGTH` in `scripts/build-changelog.ts` and the `changelog/template.md` reference. Aligns with the framework's v0.9.0 cap raise.
- **`CLAUDE.md` What's Next list** — added `security-pass` skill between the `devcheck` and `polish-docs-meta` suggestions.
- **Skill sync** — 13 framework skills refreshed: `add-tool`, `api-auth`, `api-config`, `api-errors`, `api-linter` (full portability rule reference), `api-workers` (documents the `instructions` resolver form on `WorkerHandlerOptions`), `design-mcp-server`, `field-test`, `polish-docs-meta` (+ `references/package-meta.md`, `references/readme.md`), `security-pass`, `tool-defs-analysis`.
