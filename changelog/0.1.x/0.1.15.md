# 0.1.15 — 2026-05-28

## Added

- **`MCP_HTTP_MAX_BODY_BYTES` env var** — inherited from framework 0.9.13; configurable inbound request body cap (default 1 MiB, `0` = disabled). Documented in `.env.example`.
- **`landing: { requireAuth: false }`** in `createApp()` — explicitly serves full tool/resource/prompt inventory on the landing page for this public-catalog deployment; guards against the new framework default that hides inventory behind auth when `MCP_AUTH_MODE` is `jwt` or `oauth`.

## Changed

- **`OPENFDA_API_KEY` parser** — `z.preprocess` strips MCPB placeholder strings (`${…}`) and empty strings to `undefined`, preventing strict validator crashes when the bundle passes a literal placeholder through an unset optional field.
- **`manifest.json` `user_config.openfda_api_key`** — `"default": ""` added so the install UX surfaces a blank field rather than omitting the default.
- **`package.json` keywords** — `bun`, `stdio`, `streamable-http` added; surfaced on `GET /mcp` status response via framework 0.9.12.

## Fixed

- **HTTP session-init gate** (framework 0.9.10) — stateful HTTP mode rejects non-`initialize` requests without `Mcp-Session-Id` with HTTP 400, preventing uninitialized session minting.
- **Client-error log noise** (framework 0.9.10) — 401, 403, 400, and 404 HTTP errors now log at `warning` level without stack traces instead of going through the full error handler pipeline.

## Dependencies

- **`@cyanheads/mcp-ts-core`** ^0.9.6 → ^0.9.13
- **`@biomejs/biome`** ^2.4.15 → ^2.4.16
