# 0.1.11 — 2026-05-08

## Changed

- Upgraded `@cyanheads/mcp-ts-core` `^0.7.0 → ^0.8.19` — picks up the `ctx.fail`/`ctx.recoveryFor` typed-error builder, recursive `describe-on-fields` literal exemption, the framework's antipattern guard, and the `security: true` changelog flag.
- `openfda_search_recalls` — adopted the typed error contract for the recall+non-device validation. Declares `errors: [{ reason: 'recall_endpoint_non_device', ... }]` and throws via `ctx.fail('recall_endpoint_non_device', ..., ctx.recoveryFor(...))`. Wire shape now carries `error.data.reason` and `error.data.recovery.hint` automatically.
- `openfda-service.ts` — every service-layer throw (`rate_limited`, `upstream_error`, `pagination_limit_reached`, `query_error`) now passes `data: { reason: 'X' }` to its factory so clients see the same `error.data.reason` shape they get from `ctx.fail` in handlers.
- `engines` — bumped `node ≥22 → ≥24` and `bun ≥1.2 → ≥1.3` to match the framework's runtime baseline; `Dockerfile` pinned `oven/bun:1` → `oven/bun:1.3`.
- `scripts/build-changelog.ts` — frontmatter parser learned the `security: true` flag (renders alongside `breaking` as a `🛡️ Security` badge in the rollup); badge order is fixed `Breaking · Security`.
- `CLAUDE.md` — `Errors` section rewritten around the typed contract pattern; `Context` table picks up `ctx.fail` and `ctx.recoveryFor`.
- Patch deps: `@biomejs/biome ^2.4.13 → ^2.4.14`, `@types/node ^25.6.0 → ^25.6.2`, `tsc-alias ^1.8.16 → ^1.8.17`.

## Added

- `scripts/check-framework-antipatterns.ts` — guards three SDK-coupling shortcuts (Zod `inputSchema` downgrade, post-register `inputSchema` mutation, transport-layer SDK error-text matching). Wired into `bun run devcheck` as the `Framework Antipatterns` step.
- 3 new framework skills: `api-canvas` (DataCanvas Tier 3 SQL workspace + `spillover()` helper), `api-telemetry` (OTel spans/metrics/cardinality reference), `tool-defs-analysis` (read-only audit of definition language across the surface).

## Removed

- `dev:stdio` / `dev:http` package.json scripts — unused watch-mode shortcuts; the smoke-test path is `bun run rebuild && bun run start:stdio` (or `start:http`). Stale `README.md` reference cleaned up.

## Synced

- 16 framework skills refreshed (`add-service`, `add-tool`, `api-config`, `api-context`, `api-errors`, `api-linter`, `api-utils`, `api-workers`, `design-mcp-server`, `field-test`, `maintenance`, `release-and-publish`, `report-issue-framework`, `report-issue-local`, `security-pass`, `setup`); copies in `skills/` and `.claude/skills/` regenerated.
